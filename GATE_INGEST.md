# GATE PYQ Ingestion — Roadmap

Populate the question bank with **GATE CS previous-year MCQs** (GeeksforGeeks-hosted)
and, as a side effect, make the generator produce **GATE-style** questions. The latter is
free: GATE MCQs land as `source=exemplar, verified=True`, and the serving/generation paths
are already *exemplars-first*, so they serve first **and** anchor few-shot generation.

This document is the plan of record. Status legend: ☐ pending · ◐ in progress · ✅ done.

## Why this slots into the existing pipeline

`app.scripts.ingest_exemplars` already does the hard half: LLM-parse → resolve
course/**subtopic** (67 fine-grained) → execution-verify computable keys → insert as
verified exemplars (FLAGGED if execution disputes the key). So the only new piece is a
**scraper that emits normalized items** the existing ingest consumes. No serving-path or
model changes.

## Spike findings (2026-06-11) — the approach is de-risked

The GfG quiz pages are backed by a clean JSON API; **no HTML scraping / headless browser needed**:

```
GET https://apiwrite.geeksforgeeks.org/quiz/gfg/<slug>/?page=1&page_size=65
→ { count, next, previous, results: [ {
      id, title, question(HTML), explanation(HTML),
      answers: [ { answer(HTML), correct(bool) }, … ]
  } ] }
```

- One request returns a whole paper (`page_size=65` ⇒ all 65 in one call; `next` paginates otherwise).
- The **correct answer is explicit** (`answers[].correct == true`).
- Per-question **subject is NOT tagged** → classification is delegated to the existing LLM
  mapper in `ingest`, which already assigns course + subtopic + difficulty.

**Yield, measured on GATE CS 2025 Set-1 (65 Q):** 43 four-option · 22 NAT · 10 MSQ ·
18 image/table. After dropping NAT/MSQ/image and the non-mappable subjects, realistic net is
**~15–20 usable MCQs per paper → ~180–220 across all 11 papers.**

## Decisions (locked)

1. **Answer source = key only, regenerate prose.** Scrape question + 4 options + the
   correct-answer key only. **Do not** copy GfG's explanation text. Explanations and
   distractor rationales are regenerated by the LLM (the same enrichment the generation plane
   already performs). Rationale: copyright-clean — the GATE *questions* are official public IIT
   exam papers, but GfG's *explanation prose* is their copyrighted work.
2. **Drop non-mappable subjects initially** (see note below).

> ### Scope note — intentionally dropped topics
> The following GATE subjects are **dropped on purpose** because they fall outside the
> **Dhaka University CS MSc admission** syllabus this project targets: **General Aptitude,
> Engineering Mathematics, Discrete Mathematics, Digital Logic, Compiler Design.**
> This is a scope decision, not a technical limit — the scraper sees these questions and
> simply filters them out. **If the project's scope later widens** (e.g. a general GATE-prep
> mode), reintroduce them by adding the corresponding courses to the seed taxonomy and removing
> their slugs from the drop-list in the subject filter; the rest of the pipeline already handles
> them.

## Filtering rules (a question is kept only if all hold)

- exactly **4** options, **exactly one** `correct` (drops NAT and MSQ);
- no `<img>` / `<table>` / `data:image` in question or any option (text + LaTeX model only);
- after HTML→text + math normalization, maps to a **kept** subject/course.

## Phases

- **A ✅ Spike.** Locate the answer key. JSON API + explicit `correct` flag confirmed.
- **B ✅ Scraper** `app/ingest/scrape_gfg.py` — offline, run-once. Paginated GETs over the 11
  slugs (stdlib `urllib`, rate-limited, polite UA, retry/backoff), **caches raw JSON to
  `backend/data/gate_raw/<slug>.json`**. Result: 715 raw questions cached.
- **C ✅ Normalize + filter** `app/ingest/gfg_normalize.py`. HTML→plaintext (sup/sub preserved),
  drop NAT/MSQ/image, emit `{text}` items into `backend/data/gate_ingest/<slug>.json`. Result:
  **434 candidate MCQs** (from 715).
- **D ◐ Ingest — pivoted to Claude subagents** (no external API tokens, by user request). Instead of
  the Groq/Gemini parse pipeline, parallel Claude (Sonnet) subagents read each normalized paper,
  classify to course+subtopic+difficulty (or drop out-of-scope), and regenerate explanation +
  distractor rationales, emitting the `author_questions` schema (with `examName`/`year`). Shared rules
  live in `data/gate_authored/INGEST_SPEC.md`; outputs land in `data/gate_authored/<slug>.json` and are
  loaded via `python -m app.scripts.author_questions <file>`. The Groq pipeline (B/C + the parser
  hardening) remains valid and available — it's just not used for this token-free pass.
  Pilot (gate-cs-2018): 49→30 kept, schema-validated, loaded.
- **E ✅ Verify payoff.** Serving: 4 sampled courses each serve a full 10/10 verified GATE questions
  and score 10/10. Generation anchoring: `_anchors()` now resolves few-shot examples to real GATE
  PYQs per subtopic (e.g. cache-memory → GATE 2019/2021), so `generate` produces GATE-style output.
  Verified token-free (no live `generate` run — Gemini quota preserved by request).

## Result

**254 GATE MCQs loaded** (from 715 raw → 434 candidates → 254 after Claude classification/drop), DB
now **274 questions** (254 GATE + 20 hand-authored). Per-course: computer-architecture 49,
data-structures-algorithms 55, operating-systems 32, computer-networks 33, theory-of-computation 30,
programming 28, dbms 27. AI & distributed-systems have no GATE source (author separately).
One disputed key flagged via `reviewNote` (GATE 2021 min/max-comparisons item) — official key kept.

## LLM-path learnings (live, during Phase D)

Bulk ingest surfaced provider issues; fixes landed in the shared authoring plane:

- **Gemini free tier = ~20 requests/day** (`...GenerateRequestsPerDay...FreeTier: 20`) — unusable
  for 434 calls. Added `LLM_PARSER_PROVIDER` (factory `get_parser()`); set `=groq` (~1000/day).
- **Gemini 2.5 "thinks" by default**, consuming `max_output_tokens` before the JSON completes →
  truncation. Now map `thinking=False` → `ThinkingConfig(thinking_budget=0)`.
- **Cheap models emit slightly invalid JSON** (literal newlines, unescaped quotes). Added a
  provider-agnostic `json_mode` (Gemini `response_mime_type`, Groq `response_format`), bumped
  parse budget to 8000, and hardened `extract_json` (string-aware brace match + `strict=False`).
- **Transient 503/429** — added exponential backoff to both Gemini and Groq providers.
- `ParsedQuestion.subtopicSlug` is now optional so `courseSlug="none"` drops cleanly.

> Ingestion is **not** idempotent (the LLM rewrites questionText each run, so exact-match dedup
> isn't reliable). To re-run, first delete prior ingest rows: `verifiedBy` in
> `{ingest:trusted, ingest:exec, ingest:flagged, ingest:exec-unverifiable}`.

## Legal / etiquette

GATE question papers are official public examinations published by the IITs; reproducing the
*questions* for personal exam prep is low-risk. We deliberately **do not** copy GfG's
explanation prose (Decision 1). Scrape **once**, cache locally, rate-limit, identify politely.
This honors the existing boundary documented in `app/ingest/sources.py`.
